Test Maintenance Comment Feature

Dokumen ini menjelaskan cara menguji fitur comment yang baru diimplementasikan di Maintenance Order List.

1. Test Comment Modal Opening
Test untuk memastikan comment modal dapat dibuka dari berbagai tempat di maintenance order list.
  1. Buka halaman Maintenance Order List
  2. Klik tombol comment (ikon chat) di kolom Comments
  3. Klik tombol "More Actions" (3 titik) lalu pilih "Add Comment"
Expected Result: Comment modal terbuka dengan daftar komentar yang ada (jika ada) dan form untuk menambah komentar baru.
2. Test Adding New Comment
Test untuk memastikan user dapat menambah komentar baru dengan atau tanpa attachment.
  1. Buka comment modal untuk task tertentu
  2. Tulis komentar di textarea
  3. Klik tombol "Kirim"
Expected Result: Komentar baru muncul di daftar komentar, comment count bertambah, dan form dikosongkan.
3. Test Adding Comment with Attachments
Test untuk memastikan user dapat menambah komentar dengan attachment (foto, video, file).
  1. Buka comment modal
  2. Klik tombol kamera untuk foto
  3. Klik tombol video untuk rekam video
  4. Klik tombol paperclip untuk upload file
  5. Tulis komentar dan kirim
Expected Result: Attachment berhasil diupload dan ditampilkan di komentar, file tersimpan di storage.
4. Test Comment Count Display
Test untuk memastikan jumlah komentar ditampilkan dengan benar di berbagai tempat.
  1. Lihat kolom Comments di table
  2. Lihat action menu untuk task
  3. Tambah komentar baru
  4. Refresh halaman
Expected Result: Comment count ditampilkan dengan benar dan update real-time setelah komentar ditambah.
5. Test Comment Deletion
Test untuk memastikan user dapat menghapus komentar miliknya sendiri.
  1. Buka comment modal
  2. Lihat komentar yang dibuat oleh user yang sedang login
  3. Klik tombol hapus (trash icon)
  4. Konfirmasi penghapusan
Expected Result: Komentar berhasil dihapus, comment count berkurang, dan komentar hilang dari daftar.
6. Test Comment Modal Responsiveness
Test untuk memastikan comment modal responsive di berbagai ukuran layar.
  1. Buka comment modal di desktop
  2. Resize browser window
  3. Test di mobile device atau mobile view
Expected Result: Modal responsive dan dapat digunakan dengan baik di berbagai ukuran layar.
Note: Pastikan semua endpoint API untuk comment sudah berfungsi dengan baik:
GET /api/maintenance-comments/{taskId} - Get comments
GET /api/maintenance-comments/{taskId}/count - Get comment count
POST /api/maintenance-comments - Add new comment
DELETE /api/maintenance-comments/{id} - Delete comment
Note: Fitur ini menggunakan komponen yang sama dengan maintenance order taker, jadi pastikan:
  • CommentModal.vue sudah ada dan berfungsi
  • CameraModal.vue sudah ada untuk capture foto/video
  • Database tables maintenance_comments dan maintenance_comment_attachments sudah ada
  • Storage disk public sudah dikonfigurasi dengan benar
7. Test Error Handling
Test untuk memastikan error handling berfungsi dengan baik.
  1. Test dengan network yang lambat
  2. Test dengan file yang terlalu besar
  3. Test dengan file type yang tidak didukung
  4. Test dengan API yang down
Expected Result: Error ditampilkan dengan user-friendly message, aplikasi tidak crash.
